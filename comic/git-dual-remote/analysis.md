# 内容分析 - Git 双仓库推送

## 核心信息

一个项目同时推送到 GitCode 和 GitHub 两个远程仓库, 通过 `origin` 和 `github` 两个 remote 名称区分管理。

## 目标受众

- 初学 Git 的开发者
- 需要多平台托管代码的开发者
- 对 Git remote 概念不熟悉的新手

## 价值主张

- 知识价值: 理解 Git remote 的概念, 学会多仓库管理
- 实践价值: 可直接复制命令操作
- 情感价值: 消除对 Git 多远程仓库的恐惧感

## 核心主题

| 主题 | 叙事潜力 | 视觉机会 |
|------|----------|----------|
| Git remote 概念 | 仓库像"家", 代码需要"搬家" | 两栋房子, 搬运箱子 |
| origin vs github | 两个名字对应两个地址 | 门牌号/地址标签 |
| push 操作 | 把代码"送"到远方 | 火箭/传送带/邮递 |
| pull 操作 | 从远方"取回"代码 | 收快递/钓鱼 |

## 视觉隐喻设计 (ohmsha 风格)

| 概念 | 隐喻 |
|------|------|
| Git 仓库 | 魔法传送门连接的两栋房子 |
| git remote add | 哆啦A梦掏出"任意门", 门上写着地址 |
| git push | 代码箱子通过传送带送到对应房子 |
| git pull | 从房子里用磁铁吸回代码箱子 |
| origin / github | 两扇不同颜色的任意门 (蓝色=GitCode, 黑色=GitHub) |
